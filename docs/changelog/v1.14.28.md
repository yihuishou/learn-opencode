# v1.14.28：修复 Bun 安装升级问题

**发布日期**：2026年4月27日

---

## 🎯 一句话总结

修复了使用 Bun 安装的用户运行 `opencode upgrade` 时失败的问题。

---

## 🔧 问题修复

### Bun 安装升级失败

**问题**：如果你使用 Bun 包管理器安装 OpenCode，在非项目目录（没有 `package.json` 的目录）运行 `opencode upgrade` 时会失败。

**修复**：升级命令现在正确处理 Bun 安装场景，不再依赖当前目录下存在 `package.json`。

**对你有什么影响**：
- 使用 `bun install -g opencode` 安装的用户现在可以正常升级了
- 可以在任意目录运行 `opencode upgrade`，无需先进入特定目录

---

## 📊 要不要升级？

| 情况 | 建议 |
|------|------|
| 使用 Bun 安装 OpenCode | ✅ 强烈推荐 |
| 使用 npm/brew 等其他方式 | 可以等下次 |

---

## ⬆️ 升级命令

```bash
opencode upgrade
```
